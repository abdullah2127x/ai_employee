## 📚 Architecture v2.2: Task Files Are INPUT-ONLY

From `task_template.py` comments:

```python
"""
Architecture (v2):
- Task files are INPUT-ONLY — no [PENDING] placeholders
- Claude reads task files and returns JSON decision
- Python builds output files from Claude's JSON using build_output_file()
"""
```

**What This Means:**

### ❌ OLD Way (v1 - WRONG)
```
Task file has [PENDING] placeholders
  ↓
Claude fills in the placeholders
  ↓
Same file becomes output
```

### ✅ NEW Way (v2.2 - CORRECT)
```
Task file (INPUT) → Claude reads → Returns JSON
  ↓
build_output_file() creates NEW output file
  ↓
Output file has Claude's decision
```

**Task file is NEVER modified - it's READ-ONLY input for Claude!**

## 🎯 Why File System Template is Correct

### File System Template Philosophy:

```python
# From task_template.py docstring:
"""
INPUT file that Claude reads. No [PENDING] placeholders.
"""
```

**Design Principles:**
1. **Task file = Input for Claude** (not output, not form)
2. **Claude reads → Returns JSON** (doesn't modify file)
3. **build_output_file() creates result** (separate file)
4. **No placeholders** (file is never modified)

### Gmail Template Should Follow Same Philosophy:

```python
# Task file = Input for Claude
# Contains: Email data + Gmail-specific metadata
# Claude reads → Returns JSON decision
# build_output_file() creates RESULT_*.md
```
