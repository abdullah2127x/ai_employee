# Claude Skill: File Processor

**Purpose:** Process files dropped into the vault, analyze content, categorize, and create action plans.

**Trigger:** Files in `/Processing/` folder with type `file_drop`

---

## Skill Definition

```
Skill Name: file-processor
Version: 1.0.0
Author: AI Employee Team
```

---

## Capabilities

1. **Read and analyze** any file type (PDF, DOC, TXT, images with OCR description)
2. **Extract key information** (dates, amounts, names, action items)
3. **Categorize** the file (invoice, contract, correspondence, reference, etc.)
4. **Create action plan** if processing is needed
5. **Generate summary** for the dashboard
6. **Move to appropriate folder** based on outcome

---

## Workflow

### Step 1: Read the File

```
1. Read the metadata file in /Processing/
2. Extract file path from metadata
3. Read the original file content (if text-based)
4. If binary (PDF, image), note that human review may be needed
```

### Step 2: Analyze Content

```
1. Identify document type:
   - Invoice/Billing
   - Contract/Agreement
   - Correspondence (letter, memo)
   - Report/Statement
   - Reference material
   - Unknown

2. Extract key entities:
   - Dates (deadline, due date, created date)
   - Amounts (if financial)
   - People/organizations mentioned
   - Action items identified

3. Determine urgency:
   - Has deadline approaching?
   - Marked as urgent?
   - From VIP sender?
```

### Step 3: Create Output

Based on analysis, create ONE of:

**Option A: Simple Summary (for reference files)**
```markdown
---
type: file_summary
original_file: [filename]
processed: [date]
category: reference
---

# File Summary

**Original:** [filename]
**Category:** Reference
**Processed:** [date]

## Summary

[Brief 2-3 sentence summary]

## Key Points

- [Point 1]
- [Point 2]

---
*Generated by file-processor skill*
```

**Option B: Action Plan (for files requiring work)**
```markdown
---
type: plan
objective: Process [filename]
created: [date]
status: pending
priority: [priority]
---

# Action Plan: [filename]

## Summary

[Brief summary]

## Required Actions

- [ ] Step 1: [action]
- [ ] Step 2: [action]
- [ ] Step 3: [action]

## Deadlines

- [Date]: [Milestone]

## Notes

[Any relevant notes]

---
*Generated by file-processor skill*
```

**Option C: Approval Request (if action needs approval)**
```markdown
---
type: approval_request
action: [action_type]
created: [date]
amount: [if applicable]
---

# Approval Required

**Action:** [What needs approval]

**Reason:** [Why approval is needed]

**Details:**
- [Detail 1]
- [Detail 2]

---
*Generated by file-processor skill*
```

### Step 4: Update State

```
1. Move original metadata file to /Done/
2. Move any created files to appropriate folders
3. Log the processing result
```

## Error Handling

### If file cannot be read:
```markdown
---
type: error_report
original_file: [filename]
error: Cannot read file type
---

# Processing Error

**File:** [filename]
**Error:** Unable to process binary file type

## Recommended Action

Manual review required. Please open the file directly and determine appropriate action.

---
*Generated by file-processor skill*
```

### If content is unclear:
```markdown
---
type: needs_review
original_file: [filename]
reason: Ambiguous content
---

# Manual Review Required

**File:** [filename]

**Issue:** Content analysis returned low confidence

**Suggestion:** Human review recommended to determine appropriate categorization and actions.

---
*Generated by file-processor skill*
```
